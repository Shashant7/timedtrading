# Monthly Backdrops (Phase B)

One JSON per month under `data/backdrops/<YYYY-MM>.json`. Each file is a
point-in-time snapshot of the market backdrop for that month across the Phase-B
24-ticker universe. Generated by `scripts/build-monthly-backdrop.js`.

These files are the input to Phase D's "backdrop → outcome" pairing: every
monthly backtest report is tagged with its backdrop JSON so tuning proposals
are justified by "this is what worked in this backdrop."

## Regeneration

```bash
TWELVE_DATA_API_KEY=xxx node scripts/build-monthly-backdrop.js \
  --months=2025-07,2025-08,...,2026-04 \
  --out=data/backdrops
```

Default months are `2025-07 … 2026-04`. The script caches raw TwelveData daily,
4H, and earnings pulls in `data/backdrops/.cache.json` (git-ignored); delete the
cache to force a full rebuild.

## Universe

- **Tier 1** (drives cycle + regime frequency): `SPY, QQQ, IWM, AAPL, MSFT,
  GOOGL, AMZN, META, NVDA, TSLA`.
- **Tier 2** (earnings density only, preserves existing signal): `AGQ, CDNS,
  ETN, FIX, GRNY, HUBS, IESC, MTZ, ON, PH, RIOT, SGI, SWK, XLY`.

`GOOGL` is used in place of `GOOG` because TwelveData returns the parent
Alphabet listing under the `GOOGL` symbol. `SPY / QQQ / IWM / XLY / AGQ` are
excluded from the earnings pull — ETFs and leveraged commodity products have no
meaningful earnings date.

## Schema (top-level keys)

| key | type | description |
|---|---|---|
| `schema_version` | `string` | semver-ish ("1.0" for this pass) |
| `month` | `string` | `YYYY-MM` |
| `generated_at` | `string` | builder ISO timestamp |
| `window` | `{start, end}` | inclusive calendar bounds for the month |
| `trading_days` | `string[]` | NYSE trading days in the window (from SPY daily) |
| `universe` | `{tier1, tier2}` | universe snapshot |
| `cycle` | object | market-cycle phase label + regime fractions (see below) |
| `sector_leadership` | object | per-sector RS vs SPY + rotation delta vs prior month |
| `regime_frequency` | object | Tier-1 × trading-day regime counts |
| `cross_asset_vol` | object | realized vol + VIXY/DXY/gold/oil/TLT snapshots |
| `event_density` | object | earnings + macro events in the window |
| `provenance` | object | data provider, intervals, indicator params |

### `cycle`

```
{
  "label": "uptrend" | "distribution" | "downtrend" | "accumulation" | "transitional",
  "fractions": {
    "HTF_BULL_LTF_BULL":   0.0-1.0,
    "BULL_PULLBACK":       0.0-1.0,
    "BEAR_LTF_BOUNCE":     0.0-1.0,
    "HTF_BEAR_LTF_BEAR":   0.0-1.0,
    "TRANSITIONAL":        0.0-1.0
  },
  "method": "Tier-1 × trading-day joint EMA(20) daily + 4H bias vote"
}
```

Decision rule (in order):

- `uptrend`: HTF_BULL_LTF_BULL ≥ 50 % and HTF_BEAR_LTF_BEAR < 15 %.
- `distribution`: bull + pullback ≥ 50 %, pullback ≥ 25 %, bear < 25 %.
- `downtrend`: bear ≥ 40 %, or (bear + bounce ≥ 50 % and bear ≥ 30 %).
- `accumulation`: bear < 25 %, bull < 30 %, transitional + pullback ≥ 50 %.
- Fallback: `transitional`.

### `sector_leadership`

Computed from SPDR sector ETFs (`XLK, XLF, XLE, XLY, XLI, XLV, XLP, XLU, XLRE,
XLB, XLC`) — simple first-to-last close return inside the month vs SPY's return.

```
{
  "spy_ret_pct":       number,
  "spy_prev_ret_pct":  number,
  "by_sector":         { "XLK": { "ret_pct": n, "rs_vs_spy_pct": n }, ... },
  "top":    [ {etf, label, rs_vs_spy_pct, ret_pct, rotation_delta_pp}, ... up to 3 ],
  "bottom": [ {etf, label, rs_vs_spy_pct, ret_pct, rotation_delta_pp}, ... up to 3 ]
}
```

- `rs_vs_spy_pct` = sector month return − SPY month return, in percentage
  points.
- `rotation_delta_pp` = current-month `rs_vs_spy_pct` minus prior-month
  `rs_vs_spy_pct` (positive = sector rotated into leadership).

### `regime_frequency`

```
{
  "counts":     { HTF_BULL_LTF_BULL: n, BULL_PULLBACK: n, ... },
  "per_ticker": { "SPY": { HTF_BULL_LTF_BULL: n, ... }, ... for each Tier-1 ticker }
}
```

Per Tier-1 ticker and trading day:

- `dailyBias`   = `close vs EMA(20, daily)` AND EMA slope sign.
- `intradayBias` = `close vs EMA(20, 4H)` AND EMA slope sign, using the last 4H
  bar of the day.
- EMA period = 20, slope epsilon = 1e-4 (joint-NEUTRAL if |slope| < eps).

Joint label:

- BULL × BULL → `HTF_BULL_LTF_BULL`
- BULL × BEAR → `BULL_PULLBACK`
- BEAR × BULL → `BEAR_LTF_BOUNCE`
- BEAR × BEAR → `HTF_BEAR_LTF_BEAR`
- Either NEUTRAL → `TRANSITIONAL`

This is a compact proxy for the deeper `tf_tech.D` / `tf_tech.4H` model used at
runtime; good enough to flag regime drift across months without having to
replay the full pipeline.

### `cross_asset_vol`

```
{
  "spy_realized_vol": {
    "annualized_pct":            number,   // sqrt(252) × stdev(log returns)
    "prev_month_annualized_pct": number,
    "delta_pp":                  number
  },
  "vix_proxy": {
    "symbol": "VIXY",
    "note":   "VIXY (ETF) used because ^VIX is not on the TwelveData plan",
    "avg_close", "max_close", "min_close", "prev_month_avg_close"
  },
  "move_proxy": { "symbol": null, "note": "...", "value": null },
  "dxy_proxy":  { "symbol": "UUP", "ret_pct", "prev_month_ret_pct", "trend" },
  "gold_oil_dispersion": {
    "gold_symbol": "GLD",
    "oil_symbol":  "USO",
    "gold_ret_pct", "oil_ret_pct", "dispersion_pp"
  },
  "rates_proxy": { "symbol": "TLT", "ret_pct" }
}
```

Caveats baked into the schema:

- **VIX level is not available on the TwelveData Pro plan.** `vix_proxy` uses
  VIXY closing prices — the *direction and magnitude* of VIX moves, but not the
  actual VIX points. `spy_realized_vol.annualized_pct` is provided as an
  in-sample alternative (it is directly comparable month-over-month).
- **MOVE (bond-vol) is not available either.** `move_proxy.value` is
  deliberately `null` until a proxy is wired (candidate: TLT implied vol from a
  separate source).
- **DXY** uses UUP; `trend` is `rising` / `falling` / `flat` at ±0.5 % monthly
  return thresholds.

### `event_density`

```
{
  "earnings": {
    "total_events":             n,
    "unique_tickers_reporting": n,
    "by_ticker":                { SYM: [{date, time, eps_estimate, eps_actual, surprise_pct}, ...] },
    "clusters_ge3_tickers_within_3d": [ {anchor, window_dates, tickers} ]
  },
  "macro": {
    "total_events": n,
    "events":       [ {date, name, impact} ],
    "fomc_count":   n,
    "cpi_count":    n,
    "nfp_count":    n
  },
  "density_by_date":   { "YYYY-MM-DD": n, ... },   // earnings + macro per date
  "high_density_dates": [ {date, events}, ... ]    // dates with ≥2 events
}
```

- Earnings from TwelveData `/earnings`; dates are calendar dates (no session /
  BMO / AMC inference here — that belongs to the replay gate, not the
  backdrop).
- Macro events come from `CURATED_MACRO_EVENTS` in
  `worker/market-events-seed.js` (CPI / PPI / FOMC / PCE / NFP / GDP / Retail
  Sales / ISM Manufacturing PMI / Jobless Claims).
- `clusters_ge3_tickers_within_3d` = rolling 3-day windows where ≥3 Phase-B
  tickers report. This is the signal Phase A called out for the 2025-11 R3
  drought (see `tasks/holistic-regime-calibration-plan-2026-04-17.md` §Phase
  B acceptance).

## Spot-check — Nov 2025 (plan acceptance criterion)

The plan calls for 2025-11 to flag the earnings-cluster density that caused the
R3 drought. This file shows:

- cycle: `downtrend` (HTF_BEAR_LTF_BEAR = 50 %, HTF_BULL_LTF_BULL = 36 %).
- sector leadership: Health Care / Materials / Staples **top**; Technology /
  Consumer Discretionary **bottom** — classic defensive rotation.
- earnings clusters: **five** ≥3-ticker clusters spanning Nov 3 → Nov 7, with
  up to 8 Phase-B tickers reporting in the Nov 4–6 anchor window (CDNS, ETN,
  FIX, HUBS, ON, PH, SGI, SWK). That is the 7–10 calendar-day earnings lockout
  density that the R3 fragile-entry + post-event 8h re-entry lockout compounded
  into the Nov 2025 zero-entry drought.

## Versioning

- `schema_version` is bumped whenever the top-level shape changes. Minor
  indicator-param tweaks (EMA period, slope epsilon) should update
  `provenance` but not the version.
- Builder lives at `scripts/build-monthly-backdrop.js`. Keep the curated macro
  event list in the script in sync with `worker/market-events-seed.js`.
